# Style Script YouTube — Vlog Builder
## Référence : Romain Brunel (scripts analysés x6)
## Contraste : ≠ style explication/tuto (Compte 3)

---

## PRINCIPE FONDAMENTAL

Ce style ne *explique pas* un outil. Il *documente ce que Lucas a construit et ce que ça lui rapporte*.
Le spectateur regarde quelqu'un construire son business en direct, pas un professeur.

**La différence en une phrase :**
- Style tuto : "Voilà comment tu peux faire X."
- Style vlog : "Voilà ce que J'ai fait, comment ça tourne, et ce que ça m'a rapporté."

---

## STRUCTURE NARRATIVE TYPE (20-30 min)

```
[0:00-0:30]  HOOK RÉSULTAT → révélation immédiate de ce qui a été construit
[0:30-1:30]  PROMESSE → ce que la vidéo va montrer
[1:30-3:00]  CONTEXTE PERSONNEL → qui est Lucas, pourquoi il a fait ça
[3:00-4:00]  PLACEMENT SPONSOR (si applicable) — intégré naturellement dans la démo
[4:00-17:00] TOUR DU SYSTÈME → montrer l'interface live, machine par machine
[17:00-20:00] RÉSULTATS CHIFFRÉS → ce que ça a produit concrètement
[20:00-22:00] PROCHAINES ÉTAPES → ce qui vient, teaser série
[22:00-FIN]  CTA ressource + signature verbale
```

## HOOKS — PATTERNS VALIDÉS

### Pattern 1 — Révélation d'identité changée
```
"Je ne suis pas développeur, je n'ai jamais appris à coder.
Et pourtant, ces 3 dernières semaines, j'ai construit [SYSTÈME]."
```
Fonctionnement : barrière d'entrée brisée → identification immédiate de la cible

### Pattern 2 — Révélation du système existant
```
"[Élément du système] c'est désormais [IA/automatisé].
Ça fait putaclic, mais c'est la réalité."
```
Fonctionnement : reformulation de l'identité → curiosité sur comment

### Pattern 3 — Résultat avant explication
```
"J'ai créé [N] agents IA qui travaillent 24h/24, 7j/7 pour [objectif].
Les agents ont chacun leur propre personnalité, leurs propres objectifs
et leur propre mémoire."
```
Fonctionnement : output d'abord → process ensuite

### Pattern 4 — Transformation temps/effort
```
"Il y a encore 3 mois, je faisais tout à la main — scripts, vidéos, posts.
Aujourd'hui, tout a changé grâce à Claude."
```
Fonctionnement : avant/après personnel → curiosité sur le comment

### Pattern 5 — Chiffre + contexte humain
```
"193 000 impressions LinkedIn en 28 jours.
Pendant ce temps, j'étais à la Réunion.
Tout ça, 100% automatisé avec Claude."
```
Fonctionnement : résultat concret + déplacement géographique = preuve de liberté

## VOIX ET TON

### Ce qui caractérise ce style
- **"Je/moi/mon"** en permanence — c'est son histoire, pas un tuto générique
- **Anecdotes personnelles ancrent la crédibilité** : "J'étais salarié, j'ai gagné 30 000€ au poker en 2 jours, j'ai quitté le lendemain"
- **Enthousiasme authentique** : "franchement c'est imbattable", "c'est quand même assez bluffant", "c'est Game Changer pour moi"
- **Humilité technique** : "je ne code pas", "j'avais jamais appris à coder" — permet à l'audience de s'identifier
- **Nommer Claude** (pas "l'IA") : "Cloud Code", "Claude", "l'IA" = signe qu'il est proche du produit
- **Présent de narration** : "je fais tourner", "il me génère", "ça tourne"

### Ce qu'on ne fait PAS dans ce style
- ❌ "Bonjour tout le monde, aujourd'hui on va voir..."
- ❌ "Alors, pose-toi, copie le processus..." (ton prof)
- ❌ "Pose un like, partage, abonne-toi" au milieu de la vidéo
- ❌ Slides explicatives statiques
- ❌ Analogies pédagogiques ("c'est comme une télécommande universelle...")
- ❌ "Tu/vous" comme seul mode d'adresse — alterner avec "je/moi"
- ❌ Promettre un résultat sans le montrer d'abord

## DÉMO LIVE — COMMENT MONTRER L'INTERFACE

### Règle fondamentale
Montrer l'interface de **son propre système**, pas l'interface Claude.ai ou VS Code générique.
Le spectateur regarde un produit fini qui tourne, pas une installation.

### Structure de la démo
1. **Nommer la machine/l'agent** : "Machine 01 — Script Generator"
2. **Montrer l'input** : ce que Lucas entre, ou ce qui arrive automatiquement
3. **Montrer le traitement en direct** : l'IA en train de travailler, pas juste l'output
4. **Afficher les résultats chiffrés** : vues, commentaires, leads, temps gagné
5. **Faire le lien avec la prochaine machine** : créer de la continuité

### Vocabulaire qui performe
- "mes machines" / "mon système" / "mon cockpit"
- "ça tourne 24h/24 tout seul"
- "je n'ai qu'à valider"
- "ça me prend 15 secondes maintenant" (vs "ça me prenait 1 heure")
- "il me l'a fait tout seul sans que je lui demande"

## RÉSULTATS CHIFFRÉS — OBLIGATOIRES

Chaque vidéo doit contenir au minimum 3 métriques réelles :

| Type | Exemple concret |
|------|----------------|
| Impressions | "193 000 impressions LinkedIn en 28 jours" |
| Engagement | "351 commentaires sur un seul post" |
| Croissance | "1 346 abonnés en 28 jours" |
| Temps gagné | "ça me prenait 1 heure → 15 secondes maintenant" |
| Coût | "ça me coûte 0€ de plus sur mon abonnement Max" |
| Leads | "312 commentaires = 312 emails capturés" |

Règle : **montrer le dashboard** avec les chiffres à l'écran, pas juste les dire.

## FORMAT SÉRIE — NUMÉROTATION

Ce style fonctionne en série, pas en one-shots.

```
#1 — Présentation du projet / concept
#2 — Les agents, l'architecture, comment ça tourne
#3 — Les résultats, ce qui a marché, ce qui a raté
#4 — Prochaine machine construite
```

**Intégrer systématiquement :**
- Référence à la vidéo précédente : "Comme je vous avais montré dans la dernière vidéo..."
- Teaser de la prochaine : "Dans la prochaine vidéo, je vais vous montrer..."
- CTA de renvoi : "Je vous invite à aller voir la vidéo précédente, ça apparaîtra en fin de vidéo"

## CTAs — STRUCTURE

### CTA principal (lead magnet)
```
"J'ai préparé un guide qui vous permet de faire exactement la même chose.
C'est 100% gratuit, vous retrouverez le lien dans la description."
```
**Jamais :** de prix, de vente directe, de lien vers un produit payant dans la vidéo.

### CTA commentaire (ManyChat)
```
"Commente [MOT-CLÉ] sous cette vidéo et je t'envoie [ressource] directement."
```
Le mot-clé doit être dit à l'ORAL ET être dans la description.

### CTA communauté (secondaire)
```
"Tout ce que je documente dans cette vidéo sera disponible dans [groupe/newsletter].
Le lien est en description."
```

### Signature verbale (OBLIGATOIRE — identique à chaque fin de vidéo)
```
"Je vous dis à très vite. C'était Lucas."
```
Règle STYLE-07 — jamais de variation. Toujours cette phrase exacte.

---

## PLACEMENT SPONSOR (si applicable)

Intégrer entre 1:30 et 3:00, **jamais en ouverture, jamais en fermeture**.
Le sponsor doit être un outil que Lucas utilise réellement dans la démo.

```
Structure :
1. "Et c'est là qu'intervient le partenaire de la vidéo, [NOM]."
2. Explication de l'usage réel : "C'est exactement ce que j'utilise pour..."
3. Preuve : montrer le dashboard / l'outil dans son contexte
4. Lien : "Pour tester, cliquez sur le lien en description."
```

## TEMPLATE DE SCRIPT VLOG

```
[HOOK — 0:00-0:30]
"[Résultat chiffré ou révélation d'identité]"
"[Brève explication de ce qui a été construit]"
"Dans cette vidéo, je vais te montrer [les éléments + ce qui a changé]."

[CONTEXTE PERSONNEL — 0:30-1:30]
"Il y a [durée], je faisais tout à la main : [liste des tâches]."
"Aujourd'hui, tout a changé. Voilà exactement comment."

[TOUR DU SYSTÈME — 1:30-X]
"Voici mon [nom du système]. C'est mon cockpit."
Pour chaque machine :
  → "Machine [N] — [Nom]"
  → Ce que ça prend en input
  → Ce que ça produit
  → Résultat chiffré associé
  → Temps gagné

[RÉSULTATS GLOBAUX]
"Depuis que ce système tourne, voilà ce que ça a produit :"
→ [Métriques à l'écran]

[PROCHAINES ÉTAPES]
"Ce que j'ai vu aujourd'hui, c'est seulement les premières machines."
"Dans les prochaines semaines, je vais [teaser de la suite]."

[CTA + SIGNATURE]
"J'ai préparé [ressource]. Commente [MOT-CLÉ] et je te l'envoie."
"Je vous dis à très vite. C'était Lucas."   ← signature FIXE, ne jamais modifier
```

## DIFFÉRENCES CLÉS — VLOG vs TUTO

| Critère | Style Vlog (à faire) | Style Tuto (à éviter) |
|---------|---------------------|----------------------|
| Démarrage | Résultat personnel → explication | Problème → solution |
| Sujet | "Mon système, mes résultats" | "Comment faire X" |
| Interface montrée | Son dashboard custom | Interface générique de l'outil |
| Métriques | Réelles, screenshottées | Hypothétiques / promises |
| Adresse | "J'ai fait / mon / moi" | "Tu peux / on va / voici comment" |
| Pédagogie | Aucune — montrer, pas expliquer | Analogies, slides, étapes |
| CTA likes | Jamais | Souvent ("liker, partager") |
| Longueur | 20-30 min | 10-20 min |
| Série | Oui — continuité obligatoire | Non — vidéos indépendantes |

---

## CE QUE L'AGENT YOUTUBE DOIT RETENIR

1. **Demander à Lucas ses vrais chiffres** avant d'écrire le script — sans métriques réelles, le style vlog ne fonctionne pas
2. **Le hook part toujours du résultat** — jamais du processus
3. **Pas de nom propre pour le système** — utiliser "mon système", "mes agents", "mon système d'agents Claude" (jamais "Kairos OS" ou autre nom inventé)
4. **Signature verbale OBLIGATOIRE** : "Je vous dis à très vite. C'était Lucas." — jamais de variation (règle STYLE-07)
5. **Montrer l'interface** — chaque démo doit montrer l'interface de Lucas, pas une interface générique
6. **Métriques à l'écran** — si Lucas dit un chiffre, il faut prévoir de le montrer dans la vidéo
7. **Série dès la première vidéo** — structurer comme un #1 avec teaser de la suite
